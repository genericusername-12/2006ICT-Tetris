# Proposed Role Allocation

**Status:** Draft for team discussion and approval.

This document has been prepared to support the initial team meeting. Roles and responsibilities are not final until agreed by the group.

## Member 1: Team Lead and Integration Lead

Primary responsibilities: Mitchell

- Maintain the project plan, meeting records and effort summary
- Coordinate task allocation and internal deadlines
- Maintain the GitHub repository and project workflow
- Coordinate pull requests, reviews and application integration
- Implement the splash window, main screen and exit confirmation
- Coordinate final document assembly and submission checks

## Member 2: Requirements and Configuration Lead

Primary responsibilities: Mustafa

- Draft and maintain the functional requirements
- Ensure requirements follow the precondition, event and postcondition structure
- Implement the configuration screen
- Implement field size, level, music, sound effects, AI play and extended mode controls
- Implement navigation back to the main screen
- Confirm implemented features match the functional requirements

## Member 3: Quality and Media Lead

Primary responsibilities: Luke

- Draft and maintain the non-functional requirements
- Cover usability, reliability, performance and supportability
- Implement the high-score screen
- Implement pause and resume behaviour
- Maintain the quality and testing checklist
- Coordinate the demonstration video

## Member 4: Game Architecture Lead

Primary responsibilities: Thomas

- Create the use case diagram
- Lead the core object-oriented design
- Implement the game board
- Implement tetromino creation, movement and rotation
- Implement boundary and collision behaviour
- Coordinate the use of interfaces, abstract classes and enhanced switch expressions

## Member 5: Game Logic and Testing Lead

Primary responsibilities: Harrison

- Create the activity diagram
- Implement completed-row detection
- Implement single and multiple-row clearing
- Preserve block colours after rows are cleared
- Maintain the integration test checklist
- Coordinate gameplay and navigation testing

## Shared Responsibilities

All team members are to:

- Participate in planning and design discussions
- Contribute meaningful Git commits
- Work through feature or documentation branches
- Complete work on task specific branches
- Make meaningful commits throughout the milestone
- Submit pull requests
- Review another team member's work
- Track planned and actual working time
- Test their own work before submitting it
- Supply screenshots and explanations for any coding criteria they own
- Participate in integration testing
- Contribute to the demonstration
- Submit their individual peer review
